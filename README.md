# 🛠️ Git Mastery: Laboratorio di Versioning

Questo repository è il mio spazio dedicato alla pratica e all'apprendimento di **Git**. Qui sperimento con i comandi, imparo a gestire i flussi di lavoro (workflow) e simulo situazioni reali di sviluppo collaborativo.

---

## 🎯 Obiettivi dell'Esercitazione
L'obiettivo è acquisire familiarità con il controllo di versione distribuito, focalizzandomi su:
* **Gestione dei Branch**: Creazione, navigazione e fusione di rami paralleli.
* **Risoluzione di Conflitti**: Gestione di modifiche contrastanti tra diversi rami o collaboratori.
* **Cronologia Pulita**: Utilizzo di commit descrittivi e tecniche di ripristino.

## 📜 Comandi e Concetti Esplorati

### 1. Fondamentali
* `git init`: Inizializzazione di un nuovo repository locale.
* `git status`: Monitoraggio dell'area di staging e delle modifiche.
* `git add`: Preparazione dei file per il salvataggio.
* `git commit -m "messaggio"`: Creazione di uno snapshot del progetto.

### 2. Gestione dei Rami (Branching)
* `git branch [nome]`: Creazione di nuove funzionalità in isolamento.
* `git checkout` / `git switch`: Navigazione tra le diverse versioni del progetto.
* `git merge`: Integrazione del lavoro finito nel ramo principale (`main`).

### 3. Sincronizzazione Remota
* `git remote add origin`: Collegamento tra il laboratorio locale e GitHub.
* `git push`: Caricamento dei progressi online.
* `git pull`: Aggiornamento del locale con le modifiche remote.

## 🚀 Logica di Studio
In questo repository applico la regola dei **Small & Atomic Commits**: ogni salvataggio rappresenta una singola modifica logica, con un messaggio chiaro che ne spiega il "perché" oltre al "cosa".

---

## 🛠️ Strumenti Utilizzati
- **Git Bash / Terminal**: Per l'esecuzione dei comandi a riga di comando.
- **GitHub**: Per l'hosting e la visualizzazione del grafo dei commit.
- **Visual Studio Code**: Editor per la modifica dei file e la risoluzione dei conflitti grafici.
